TestPilot AI — Technical Breakdown
Architecture

Frontend: React dashboard — test flows, run history, and alerting UI.

Backend: Node.js/Express — Playwright automation engine, REST API for triggering and managing test runs, persistent storage for artifacts and logs.

Infra: Railway for deployment. Shopify deploy webhooks trigger automated runs on each deployment.

What I Built

Designed the Playwright-based automation runner to simulate real user checkout flows. Built backend APIs to trigger runs manually or via webhook and retrieve run history. Implemented Shopify-specific checkout handling including product variant selection via API and cart injection before UI validation. Built a run classification system to detect checkout success, UI failures, and navigation errors. Added retry logic and failure thresholds to reduce false alerts. Integrated screenshot and log artifact generation for debugging.

Key Challenges

Unreliable UI across themes: Built a hybrid approach using API-based cart insertion and flexible role/text-based selector fallbacks.

False positives: Implemented retry system with delay — only alert after consecutive failures.

Partial page loads: Combined URL checks, DOM content validation, and form detection to distinguish real failures from transient issues.

Outcome: Reduced manual checkout testing from ~30 min per deploy to fully automated runs with actionable failure insights.

LegalBill AI — Technical Breakdown
Architecture

Mobile: React Native / Expo — voice input, matter selection, billing entry output with copy-to-clipboard.

Backend: Node.js API on Railway — handles transcription input, OpenAI integration, and prompt processing.

AI Pipeline: Speech → transcription → LLM formatting into structured billing entries (0.1 hr increments, action-oriented phrasing).

Infra: Railway (backend), EAS (Android APK builds), Vercel (landing page). Clio-compatible workflows.

What I Built

Developed the mobile-first voice interface for capturing work logs. Built backend API to process transcription input and generate structured billing entries. Designed prompt templates to normalize messy voice input into professional legal billing language with consistent formatting. Implemented domain-specific rules including action-oriented phrasing and standardized entry structure. Kept OpenAI keys server-side by decoupling the app from any direct AI calls.

Key Challenges

Unstructured voice input: Designed prompt templates to enforce consistent formatting and normalize inconsistent speech.

Legal billing standards: Generic AI outputs aren't billable — fine-tuned prompts for legal tone and added formatting validation.

Frictionless UX: Optimized the flow to speak → process → display instantly with minimal steps.

Outcome: Eliminates end-of-day time reconstruction — structured, ready-to-use billing entries immediately after input.

Secure Serverless Web App — Technical Details
Architecture

Frontend: Static site hosted on S3.

Backend: AWS Lambda (Node.js) behind API Gateway (HTTP API).

Data: DynamoDB for NoSQL storage with stateless access patterns.

Auth: AWS Cognito with JWT-based authentication via API Gateway authorizer.

Infra: Fully provisioned via Terraform — API Gateway routes, Lambda functions, DynamoDB tables, Cognito user pool.

What I Built

Provisioned the complete infrastructure using Terraform. Implemented Lambda functions for creating and retrieving data. Integrated JWT authentication via Cognito and API Gateway authorizer. Designed stateless backend functions with DynamoDB access patterns for efficient reads and writes.

Key Challenges

Lambda event handling: Standardized request parsing based on API Gateway v2 event structure to handle inconsistencies.

Auth integration: Configured Cognito issuer and JWT validation, debugged unauthorized responses through API Gateway.

Outcome: Fully serverless, scalable architecture with infrastructure reproducible via code and secure authenticated API access.

IoT Inverter Monitoring System — Technical Details
Architecture

Device Communication: MQTT over AWS IoT Core — publish/subscribe model for real-time telemetry streaming.

Backend: AWS Lambda functions triggered by IoT rules to process incoming device data.

API Layer: API Gateway exposing REST endpoints for data consumers.

Storage: DynamoDB for low-latency telemetry storage and retrieval.

What I Built

Simulated real-time inverter telemetry using MQTT topics. Configured AWS IoT Core for device communication and designed the MQTT topic structure for efficient routing. Built Lambda functions to process incoming data and store telemetry in DynamoDB. Exposed REST endpoints via API Gateway for retrieving device data.

Key Challenges

Real-time streaming: Used MQTT publish/subscribe model to handle continuous device-to-cloud data flow efficiently.

Event-driven architecture: Structured Lambda triggers from IoT rules for reliable, asynchronous processing.

Outcome: Real-time monitoring system with a scalable serverless backend demonstrating production-grade IoT and event-driven patterns.
